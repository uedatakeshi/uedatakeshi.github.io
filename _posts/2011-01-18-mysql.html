--- layout: post title: MySQLで、ログイン用のメールアドレスなんかはバイナリー属性にしてテーブルを作る date: '2011-01-18T12:46:00.001+09:00' author: 上田 剛 tags: - MySQL modified_time: '2011-01-18T12:46:36.260+09:00' blogger_id: tag:blogger.com,1999:blog-2052559951721308867.post-4028717695956950996 blogger_orig_url: http://uedatakeshi.blogspot.com/2011/01/mysql.html --- なんかなー、デフォルトで大文字小文字区別しないってことが問題なんじゃないの？
って気はするけどなー。
普通にこういうテーブルを作る。
CREATE TABLE samples (
  id INT,
  email VARCHAR(255)
);

emailは大文字でも小文字でもヒットする。
バイナリー属性を付ける。
CREATE TABLE samples (
  id INT,
  email VARCHAR(255) binary
);
emailは大文字は大文字、小文字は小文字でヒットする。
show で見るとこんなになってる。

CREATE TABLE `samples` (
  `id` int(11) default NULL,
  `email` varchar(255) character set utf8 collate utf8_bin default NULL
) ENGINE=MyISAM DEFAULT CHARSET=utf8
